# draw.io（diagrams.net）調査レポート

作成日: 2026-02-21

---

## 1. 概要

### draw.io（diagrams.net）とは

draw.io（正式名称: diagrams.net）は、Webブラウザ上で動作する無料のダイアグラム作成ツールです。JGraph Ltd（英国）が開発・提供しており、ソースコードはApache License 2.0のもとGitHubで公開されています。

主な特徴:

- **完全無料**: 基本機能は無料で利用可能、アカウント登録不要
- **オープンソース**: GitHub上で公開（https://github.com/jgraph/drawio）
- **セキュリティファースト**: 図のデータをサーバーに保存しない設計
- **ノーベンダーロック**: 標準XMLフォーマットでファイルを保存
- **Microsoft Visioの代替**: TechRadar・PCMagが代替ツールとして評価

## 2. 機能一覧

### 図の種類

| カテゴリ | 具体的な図の種類 |
|---------|----------------|
| ビジネス | フローチャート、組織図、プロセス図、BPMN |
| ソフトウェア | UML（クラス図、シーケンス図、ユースケース図等）、ER図 |
| インフラ | ネットワーク図、AWS/Azure/GCP構成図 |
| 思考整理 | マインドマップ、ツリー図 |
| その他 | ガントチャート、フロアプラン、電気回路図、インフォグラフィック |

### 主要機能

- **ドラッグ&ドロップ編集**: 豊富な形状ライブラリから図形をドラッグして配置
- **リアルタイム協働編集**: 複数ユーザーが同時に編集可能
- **AIによる図の生成**: Generate Toolで自然言語から図を自動生成（インフラ図、Mermaid図等）
- **自動レイアウト**: 接続された図形を自動整列
- **レイヤー管理**: 複数レイヤーで図を構造化
- **カスタム形状**: 独自の形状ライブラリを作成・インポート可能
- **豊富なエクスポート形式**: PNG、SVG、PDF、HTML、XML等
- **テンプレートライブラリ**: 業種別・用途別テンプレートを多数収録
- **ダークモード**: 自動または手動でカラー切替
- **Network 2025形状ライブラリ**: 最新のネットワーク機器形状（影付きスタイル等）
- **オフライン動作**: デスクトップアプリでインターネット不要

## 4. 対応プラットフォーム・データ保存

### 対応プラットフォーム

| 種類 | 詳細 |
|------|------|
| Web版 | app.diagrams.net（Chrome、Firefox、Edge、Safari等の主要ブラウザ対応） |
| デスクトップ版 | Windows、macOS、Linux（Electron製、無料） |
| モバイル | ブラウザ経由で利用可能（専用アプリなし） |
| VS Code拡張 | hediet.vscode-drawio（サードパーティ製） |

### データ保存先

| 保存先 | 説明 |
|--------|------|
| ローカルファイル | デバイス上に.drawioまたは.xmlとして保存 |
| Google Drive | Googleアカウントと連携して保存・共有 |
| Microsoft OneDrive | Microsoftアカウントと連携 |
| Dropbox | Dropboxアカウントと連携 |
| GitHub / GitLab | リポジトリ内にバージョン管理しながら保存 |
| ブラウザストレージ | 一時的なブラウザ内保存（推奨しない） |

**セキュリティポイント**: draw.ioはダイアグラムデータをサーバーに送信・保存しません。データはユーザーが選択した保存先（ローカル or クラウドストレージ）にのみ保存されます。機密情報を扱う法律事務所にとって、この設計は安心材料です。

## 5. 導入方法

### 5-1. Web版（app.diagrams.net）の使い方

1. ブラウザで https://app.diagrams.net/ にアクセス
2. 保存先を選択（「ローカルに保存」「Google Driveに保存」等）
3. テンプレート選択または「白紙から作成」を選ぶ
4. 左パネルの形状ライブラリからドラッグ&ドロップで図を作成
5. 完成後、File > Save As でファイルを保存、またはFile > ExportでPDF/PNG等に書き出し

**アカウント登録不要**でその場から使い始められます。

### 5-2. デスクトップアプリのインストール手順

**Windows / Mac / Linux共通**:

1. https://github.com/jgraph/drawio-desktop/releases にアクセス
2. 最新バージョン（2025年11月時点: v29.0.3）のインストーラーをダウンロード
   - Windows: `.exe`インストーラー または `.msi`
   - macOS: `.dmg`ファイル
   - Linux: `.AppImage` または `.deb`
3. ダウンロードしたファイルを実行してインストール
4. アプリを起動するとWeb版と同じUIが使用可能（完全オフライン動作）

**メリット**: インターネット接続不要、ローカルファイルとして保存、セキュリティが高い

### 5-3. VS Code拡張のセットアップ

1. VS Codeを開き、拡張機能マーケットプレイスを検索
2. `hediet.vscode-drawio`（Draw.io Integration）を検索してインストール
3. `.drawio`または`.dio`拡張子のファイルをVS Codeで開くと、自動的にdraw.ioエディターが起動
4. ソースコードと同じリポジトリでダイアグラムを管理可能（Gitでバージョン管理）

**注意**: これはサードパーティ製の非公式拡張ですが、GitHubで広く使われています。

### 5-4. Google Drive（Google Workspace）との連携設定

**方法1: Google Workspace Marketplace経由**

1. Googleアカウントでログイン
2. Google Workspace Marketplace（https://workspace.google.com/marketplace）で「draw.io」を検索
3. 「インストール」をクリック → 「続行」 → アカウント選択
4. draw.ioへのGoogleドライブアクセス許可を「許可」
5. インストール完了後、Google DriveのUIから直接draw.ioが利用可能

**方法2: app.diagrams.netからの連携**

1. https://app.diagrams.net/ にアクセス
2. 保存先として「Google Drive」を選択
3. Googleアカウントでの認証を行う
4. 以降、作成した図はGoogleドライブに自動保存

**連携後の機能**: 図の自動保存、バージョン履歴管理、Googleドライブ上でのファイル共有

### 5-5. Notion連携の方法

draw.ioはNotionとの連携が可能です（主にChrome拡張経由）:

**Chrome拡張「draw.io for Notion」を使う方法**:

1. Chrome ウェブストアで「draw.io for Notion」を検索してインストール
2. Chromeの拡張機能アイコンからピン留めする
3. Notionのページを開き、拡張機能アイコンをクリック → 「Insert Diagram」
4. draw.ioエディターが起動し、図を作成
5. 完成後に保存するとNotionページに図が埋め込まれる

**制限事項**: 1ページに埋め込み可能な図は1枚のみ。複数ページの図が必要な場合はGoogle Drive等で管理が必要。

**直接埋め込み方法（別途）**: draw.ioで図を作成後、File > Embed > Notionからエクスポートして埋め込む方法も利用可能。

## 6. 弁護士業務での活用方法

### 6-1. 訴訟進行フローチャート

**用途**: 事件の訴訟手続きの流れを視覚化し、依頼人への説明や事務所内の業務管理に活用。

**具体的な作成例**:
- 民事訴訟の流れ（訴状提出 → 答弁書 → 第1回口頭弁論 → 証拠調べ → 判決）
- 保全処分申立て（仮差押・仮処分）の手続きフロー
- 刑事事件フロー（逮捕 → 勾留 → 起訴/不起訴 → 公判 → 判決）
- 労働審判手続きのフロー

**draw.ioでの実装**: フローチャートテンプレートを選択し、手続きの各ステップを「処理」形状（長方形）、分岐点を「判断」形状（ひし形）で表現。依頼人向けにはシンプルな色分けで見やすく仕上げる。

**効果**: 依頼人が手続きの全体像を一目で把握でき、説明時間を短縮。次のステップや期限も明確化。

---

### 6-2. 案件関係者の相関図（ステークホルダーマップ）

**用途**: 事件関係者（当事者、証人、代理人、関連機関）の関係性を整理・可視化。

**具体的な作成例**:
- 相続案件: 被相続人を中心に法定相続人・受遺者・遺言執行者の関係
- M&A/企業再編: 親会社・子会社・関連会社・株主・債権者の相関
- 不動産トラブル: 売主・買主・仲介業者・金融機関・登記名義人の関係
- 刑事事件: 被告人・被害者・共犯者・目撃者・捜査機関の関係

**draw.ioでの実装**: 人物を楕円や人物アイコン形状で表し、矢印で関係性（「雇用」「親族」「取引先」等）を表現。レイヤー機能を使い、当初の関係と現在の関係を分けて管理。

**効果**: 複雑な利益相反チェックが容易になる。新任スタッフへの引き継ぎ資料として活用可能。

---

### 6-3. 契約書レビューのチェックリスト/フロー

**用途**: 契約書審査の手順を標準化し、抜け漏れを防ぐ。

**具体的な作成例**:
- 契約書審査フロー: 受領 → 種別確認 → 必須条項チェック → リスク評価 → 修正案作成 → 承認
- 重要条項チェックリスト（損害賠償条項、解除条項、管轄条項、秘密保持条項等）
- 業種別リスク評価マトリクス（リスクの大小×対応難易度）
- 契約類型ごとのレビューテンプレート（業務委託、売買、ライセンス等）

**draw.ioでの実装**: BPMN（ビジネスプロセスモデリング表記）シェイプライブラリを使い、担当者ごとの泳ぎ（スイムレーン）で役割分担を明確化。チェックボックス型のフロー図で未確認項目を一目で把握。

**効果**: ジュニア弁護士・パラリーガルが標準的な品質でレビューを実施できる。事務所全体の審査品質の均一化。

---

### 6-4. 法的論点の整理（マインドマップ）

**用途**: 案件の争点や法的論点を整理し、主張立案・反論検討に活用。

**具体的な作成例**:
- 訴訟戦略マインドマップ: 中心に「請求原因」を置き、各争点・証拠・反論をブランチで展開
- 判例調査の整理: 中心テーマから、有利判例・不利判例・区別論を分岐
- 解釈論の整理: 条文解釈に関する学説・判例・通説をツリー構造で整理
- 法改正影響分析: 改正前後の規定を比較しながら依頼人への影響を整理

**draw.ioでの実装**: マインドマップテンプレートを選択。中心ノードから分岐し、色分けで「有利」「不利」「未確認」を区別。PDFエクスポートで弁論メモとして印刷。

**効果**: 論点の見落とし防止。チームでのケース検討ミーティングで議論を整理しやすくなる。

---

### 6-5. 業務プロセスの可視化（事務所DX推進）

**用途**: 法律事務所内の業務フローを可視化し、非効率な工程を発見・改善する。

**具体的な作成例**:
- 受任フロー: 初回相談 → 受任判断 → 委任契約 → 着手金請求 → 事件番号発番 → ファイル作成
- 書類管理フロー: 受領 → スキャン → フォルダ分類 → システム登録 → 原本保管場所記録
- 請求・入金管理: 報酬算定 → 請求書作成 → 送付 → 入金確認 → 領収書発行
- 期日管理フロー: 期日設定 → カレンダー登録 → リマインダー送信 → 準備確認 → 出廷

**draw.ioでの実装**: BPMN形状または標準フローチャートで、各業務ステップ・担当者・使用システムを記載。ボトルネックになっている工程を赤くハイライト。

**効果**: 属人化した業務の標準化。スタッフ研修資料として使用。業務改善の優先順位付けが容易になる。

---

### 6-6. 案件タイムライン（時系列図）

**用途**: 事件の時系列を整理し、主張の一貫性確認や陳述書作成に活用。

**具体的な作成例**:
- 紛争発生の経緯（契約締結→債務不履行→催告→提訴）の時系列
- 刑事事件の犯行前後の行動タイムライン
- 不貞行為案件における証拠の発見日時系列
- 労働事件（解雇・ハラスメント）における出来事の時系列

**draw.ioでの実装**: 水平タイムライン形状を使用。重要イベントをマーカーで強調。Googleドライブに保存し、弁護団全員がリアルタイムで更新・共有。

**効果**: 主張と証拠の時系列的な一貫性を視覚的に確認できる。陳述書の下書きとして活用可能。

---

### 6-7. 企業法務：組織図と意思決定フロー

**用途**: 顧問先企業の組織構造や意思決定プロセスを可視化し、法的リスクの所在を特定。

**具体的な作成例**:
- 顧問先の組織図（株主総会→取締役会→各部門）と法的権限の範囲
- 契約締結権限の委任フロー（誰が何円までの契約に署名できるか）
- コンプライアンス報告ライン（問題発生→内部報告→弁護士への相談フロー）
- グループ企業の資本関係図（親子会社・関連会社の関係）

**draw.ioでの実装**: 組織図テンプレートを使用。権限範囲を色分け（赤=要取締役会承認、緑=部門長権限等）。

**効果**: 顧問先との打ち合わせ時に組織の実態を素早く把握。契約書の代理権・署名権限の確認が容易になる。

## 7. まとめ・推奨事項

### 総評

draw.io（diagrams.net）は、**弁護士業務に最適な無料のダイアグラムツール**です。特に以下の点が法律事務所での利用に適しています:

| 評価軸 | 評価 | 理由 |
|--------|------|------|
| コスト | ◎ | 完全無料（Web版・デスクトップ版） |
| セキュリティ | ◎ | データをサーバーに送信しない設計 |
| 使いやすさ | ○ | テンプレートが豊富で初心者でも使いやすい |
| 連携性 | ○ | Google Drive、Notion等と連携可能 |
| オフライン対応 | ◎ | デスクトップアプリで完全オフライン利用可能 |
| 機密性 | ◎ | アカウント登録不要、クラウド保存はオプション |

### 法律事務所向け推奨導入方法

**ステップ1**: まず https://app.diagrams.net/ でWeb版を試用（登録不要）

**ステップ2**: 機密情報を扱う場合はデスクトップアプリをインストールし、ローカル保存で運用

**ステップ3**: 事務所全体で共有する場合はGoogleワークスペースと連携し、チームでリアルタイム共同編集

**ステップ4**: すでにNotionを使っている場合は、Chrome拡張でNotion内に図を直接埋め込む

### 優先的に活用すべき機能

1. **フローチャート**: 訴訟手続きの説明・業務プロセス整理（学習コスト最低）
2. **相関図**: 関係者マップ・組織図の作成（視覚的効果が高い）
3. **マインドマップ**: 法的論点の整理・チームでのブレインストーミング
4. **タイムライン**: 事件の時系列整理（陳述書・準備書面作成の補助として）

### 競合ツールとの比較

| ツール | 料金 | 弁護士業務での優位性 |
|--------|------|---------------------|
| draw.io | 無料 | セキュリティ、コスト、豊富な図の種類 |
| Lucidchart | 有料 | 洗練されたUI（ただし高コスト） |
| Microsoft Visio | 高額 | Office連携（ただし非常に高コスト） |
| Miro | 一部無料 | リアルタイム協働に強い |

**結論**: 法律事務所が初めてダイアグラムツールを導入する場合、draw.ioは**コスト・セキュリティ・機能のバランスが最も優れた選択肢**です。無料で利用でき、機密データをサーバーに送信しない設計は、弁護士の守秘義務の観点からも優れています。
