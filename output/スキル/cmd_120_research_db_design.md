# 1. Notion 案件データベース設計

弁護士業務の特性（案件管理・書面管理・期日管理・通信履歴の統合管理）に最適化したNotion DBシステムを設計。

## 1.1 テーブル（DB）構成

### DB1: 案件マスタ（Cases）

**目的**: 全案件の中核となるマスタDB。他の全DBがここを参照する。

| プロパティ名 | タイプ | 説明 | 設定値・制約 |
|-------------|--------|------|-------------|
| 案件ID | Title | 一意識別子 | `2026_001` 形式（年_連番） |
| 案件名 | Text | 短い案件名 | 例: `山田太郎_離婚調停` |
| 受任日 | Date | 受任日 | - |
| 終了日 | Date | 終結日 | - |
| 案件種別 | Select | 案件分類 | 離婚・相続・交通事故・債務整理・刑事・その他 |
| ステータス | Status | 進行状況 | 相談中・受任・進行中・終結・アーカイブ |
| 担当弁護士 | Person | 主担当者 | - |
| 書面数 | Rollup | 紐づく書面数 | 書面DB → Count all |
| 次回期日 | Rollup | 最も近い期日 | 期日DB → 日時 → Earliest |
| 最新通信日 | Rollup | 最終通信日 | 通信記録DB → 日時 → Latest |
| 備考 | Text | 補足情報 | 長文可 |
| Drive案件フォルダ | URL | フォルダリンク | Google Driveの案件フォルダURL |

**ビュー構成例**:

- **案件一覧（テーブルビュー）**: ステータス別フィルター、受任日降順ソート
- **進行中案件（ボードビュー）**: ステータスでグルーピング（カンバン方式）
- **今月終結予定（フィルタードビュー）**: 終了日が今月の案件のみ表示

---

### DB2: 当事者（Parties）

**目的**: 依頼者・相手方・相手方代理人など、案件関係者を一元管理。

| プロパティ名 | タイプ | 説明 | 設定値・制約 |
|-------------|--------|------|-------------|
| 当事者名 | Title | フルネーム | - |
| 案件 | Relation | 紐づく案件 | → 案件マスタDB |
| 続柄/立場 | Select | 関係性 | 依頼者・相手方・相手方代理人・証人・その他 |
| 電話番号 | Phone | 連絡先（電話） | - |
| メールアドレス | Email | 連絡先（メール） | - |
| 住所 | Text | 郵送先 | - |
| 備考 | Text | 補足情報 | 家族構成・注意事項等 |

**ビュー構成例**:

- **依頼者一覧（テーブルビュー）**: 続柄=依頼者でフィルター
- **相手方代理人一覧（テーブルビュー）**: 続柄=相手方代理人でフィルター

---

### DB3: 書面（Documents）

**目的**: 訴状・準備書面・証拠（甲乙号証）・陳述書など、すべての書面を管理。Google Driveと連携。

| プロパティ名 | タイプ | 説明 | 設定値・制約 |
|-------------|--------|------|-------------|
| 書面名 | Title | 書面の名称 | 例: `原告第一準備書面` |
| 案件 | Relation | 紐づく案件 | → 案件マスタDB |
| 種別 | Select | 書面分類 | 訴状・答弁書・準備書面・甲号証・乙号証・陳述書・裁判所文書・その他 |
| 証拠番号 | Text | 証拠番号 | 例: `甲1`, `乙3-2` |
| 提出日 | Date | 提出日（空欄可） | - |
| 受領日 | Date | 受領日（空欄可） | - |
| Driveリンク | URL | ファイル直リンク | Google DriveのファイルURL |
| 備考 | Text | 補足情報 | 内容要旨・作成者など |

**ビュー構成例**:

- **書面一覧（テーブルビュー）**: 提出日降順ソート
- **案件別書面（グループドビュー）**: 案件でグルーピング
- **証拠一覧（フィルタードビュー）**: 種別=甲号証 or 乙号証でフィルター、証拠番号でソート
- **書面タイムライン（タイムラインビュー）**: 提出日/受領日でタイムライン表示

---

### DB4: 期日（Hearings）

**目的**: 口頭弁論・調停・打合せなど、すべての期日をカレンダー管理。

| プロパティ名 | タイプ | 説明 | 設定値・制約 |
|-------------|--------|------|-------------|
| 期日名 | Title | 期日の名称 | 例: `第3回口頭弁論` |
| 案件 | Relation | 紐づく案件 | → 案件マスタDB |
| 期日種別 | Select | 期日分類 | 口頭弁論・弁論準備・調停・示談交渉・打合せ・その他 |
| 日時 | Date | 期日日時 | 時刻も含める |
| 場所 | Text | 裁判所名・法廷番号 | 例: `東京地裁401号法廷` |
| 備考 | Text | 補足情報 | 持参書類・注意事項など |
| 完了 | Checkbox | 終了フラグ | 期日終了後にチェック |

**ビュー構成例**:

- **期日カレンダー（カレンダービュー）**: 日時でカレンダー表示
- **今月の期日（フィルタードビュー）**: 今月の期日のみ、完了=未チェックでフィルター
- **案件別期日（グループドビュー）**: 案件でグルーピング

---

### DB5: 通信記録（Communications）

**目的**: メール・電話・LINEなど、すべての対外通信履歴を記録。依頼者対応の証跡管理。

| プロパティ名 | タイプ | 説明 | 設定値・制約 |
|-------------|--------|------|-------------|
| 通信タイトル | Title | 通信の件名 | 例: `山田様からの電話相談` |
| 案件 | Relation | 紐づく案件 | → 案件マスタDB |
| 相手方 | Relation | 通信相手 | → 当事者DB |
| 通信種別 | Select | 通信手段 | メール・電話・LINE・FAX・書面受領・その他 |
| 日時 | Date | 通信日時 | - |
| 要旨 | Text | 概要メモ | 長文可（数行で要点をまとめる） |
| 原文リンク | URL | 元データへのリンク | Notionページ or Google DriveリンクorGmailスレッドURL |
| 要返信 | Checkbox | 返信必要フラグ | 未対応タスクの可視化 |

**ビュー構成例**:

- **通信履歴（テーブルビュー）**: 日時降順ソート
- **未処理通信（フィルタードビュー）**: 要返信=チェック済みでフィルター
- **案件別通信（グループドビュー）**: 案件でグルーピング

- **最新通信日**: 通信記録DB → 日時 → Latest → 最後に通信した日付を自動表示

## 1.3 テンプレート設計

### テンプレート1: 新規案件登録テンプレート

**目的**: 受任時の入力作業を効率化。

**テンプレート内容**:

1. **案件マスタ**: 案件ID（自動採番）、案件名、受任日、案件種別、担当弁護士を入力
2. **当事者DB**: 依頼者の基本情報（氏名・連絡先）を自動作成
3. **期日DB**: 初回打合せ日時を自動作成
4. **通信記録DB**: 相談受付時の記録を自動作成

**実装方法**: Notionのテンプレートボタン機能で、案件マスタのページに配置。

## 1.4 Notionでの作成手順

### ステップ1: 5つのデータベースを作成

1. Notionで新規ページを作成（例: `案件管理システム`）
2. `/database` でテーブル型データベースを5つ作成:
    - `案件マスタ`
    - `当事者`
    - `書面`
    - `期日`
    - `通信記録`

### ステップ2: プロパティの設定

各DBの「プロパティを追加」から、上記のテーブル定義に従ってプロパティを作成。

### ステップ3: リレーションの設定

1. `当事者`DB → プロパティ追加 → Relation → `案件マスタ` を選択
2. `書面`DB → プロパティ追加 → Relation → `案件マスタ` を選択
3. `期日`DB → プロパティ追加 → Relation → `案件マスタ` を選択
4. `通信記録`DB → プロパティ追加 → Relation → `案件マスタ` と `当事者` を選択

### ステップ4: ロールアップの設定

`案件マスタ`DBで:

- プロパティ追加 → Rollup → `書面` → Count all
- プロパティ追加 → Rollup → `期日` → `日時` → Earliest
- プロパティ追加 → Rollup → `通信記録` → `日時` → Latest

### ステップ5: ビューの作成

各DBの右上「+」から新しいビューを追加。フィルター・ソート・グルーピングを設定。

- **テンプレートフォルダ**: 使い回すひな形を一元管理
- **共通フォルダ**: 案件に紐づかない全事務所共通資料

## 2.2 ファイル命名規則

### 基本形式

```
YYYYMMDD_種別番号_書面名.拡張子
```

### 命名例

| 書面種別 | ファイル名例 |
|---------|-------------|
| 甲号証 | `20260212_甲1_売買契約書.pdf` |
| 甲号証（枝番） | `20260212_甲1-2_契約書別紙.pdf` |
| 乙号証 | `20260215_乙5_診断書.pdf` |
| 準備書面 | `20260220_準備書面1_原告第一準備書面.pdf` |
| 訴状 | `20260210_訴状_損害賠償請求.pdf` |
| 答弁書 | `20260218_答弁書.pdf` |
| 陳述書 | `20260225_陳述書_依頼者.pdf` |
| 裁判所文書 | `20260301_第2回期日調書.pdf` |

**命名規則のポイント**:

- **日付プレフィックス**: 作成日・受領日を先頭に配置（時系列ソートが容易）
- **種別番号**: 証拠番号や準備書面番号を明記
- **枝番対応**: `甲1-2` のようにハイフンで枝番を表現
- **拡張子**: PDF原則（Wordは `.docx`、Excelは `.xlsx`）

## 2.3 Notion ↔ Drive 連携方法

### 手動連携（基本運用）

1. Google DriveにファイルをアップロードGoogle Driveの案件フォルダに書面を配置
2. ファイルのリンクをコピー:
    - ファイルを右クリック → 「リンクを取得」 → URLをコピー
3. Notion書面DBに登録:
    - Notion書面DBに新しいページを作成
    - `Driveリンク`プロパティにURLをペースト
    - 書面名・種別・証拠番号などを入力

### 自動連携（n8n活用案）

**ワークフロー概要**:

```
[Google Drive Trigger]
 ↓ (File Created in 案件フォルダ)
[Function Node: ファイル名パース]
 ↓ (YYYYMMDD_種別番号_書面名 から種別・証拠番号を抽出)
[Notion Node: 書面DB作成]
 ↓ (書面名・種別・証拠番号・DriveリンクをNotionに自動登録)
[Google Chat通知] (optional)
 ↓ (「新しい書面が登録されました」)
```

**n8n実装のポイント**:

- **トリガー**: Google Drive Trigger (`File Created`) で案件フォルダを監視
- **Function Node**:
    - ファイル名から `YYYYMMDD_種別番号_書面名` を正規表現でパース
    - 種別（甲号証・乙号証・準備書面等）を判定
    - 証拠番号を抽出（例: `甲1`, `乙3-2`）
- **Notion Node**:
    - 書面DBに新しいページを作成
    - `書面名`: パースしたファイル名
    - `種別`: 判定した種別
    - `証拠番号`: 抽出した証拠番号
    - `Driveリンク`: トリガーから取得したファイルURL
    - `案件`: ファイルパスから案件ID（`2026_001`）を抽出し、Notionの案件マスタとリレーション
- **Google Chat通知**: 家老・殿への通知（任意）

**注意事項**:

- 命名規則を厳守しないと自動パースが失敗する → 運用時にルールを徹底
- 案件フォルダ名とNotion案件IDの一致が必須
- Drive API認証とNotion API認証が必要

## 2.4 Drive運用の補足ガイド

### フォルダ権限設定

- **事務所全体共有**: `法律事務所/` フォルダを事務所メンバー全員に共有（編集権限）
- **外部共有禁止**: `案件/` フォルダ配下は外部共有を無効化（誤送信防止）

### 定期バックアップ

- Google DriveのバックアップをGoogle Takeoutで定期実行（月1回推奨）
- 重要案件は終結時にローカルHDDにもバックアップ

### アーカイブ運用

- 終結から3年経過した案件: `案件_アーカイブ/` フォルダに移動
- Notion案件マスタのステータスを「アーカイブ」に変更

# 3. 統合運用フロー（例: 新規受任時）

## 新規案件受任のステップ

### ステップ1: Notion案件マスタに登録

- 「新規案件登録テンプレート」を使用
- 案件ID（`2026_004`）、案件名、受任日、案件種別を入力
- 担当弁護士を選択

### ステップ2: Google Driveに案件フォルダ作成

- `法律事務所/案件/` 配下に `2026_004_田中次郎_債務整理/` フォルダを作成
- サブフォルダ（01〜08）を一括作成（テンプレートフォルダからコピー推奨）

### ステップ3: Notion案件マスタにDriveリンク追加

- 作成した案件フォルダのURLをコピー
- Notion案件マスタの `Drive案件フォルダ` プロパティにペースト

### ステップ4: 当事者情報を登録

- Notion当事者DBに依頼者を登録（氏名・連絡先・続柄=依頼者）
- 案件マスタとリレーション

### ステップ5: 初回打合せ期日を登録

- Notion期日DBに初回打合せを登録（日時・場所・期日種別=打合せ）
- 案件マスタとリレーション
