## 18. legal-office-research（法律リサーチ総合スキル）— 17点

### (1) 概要

法律に関わるリサーチ業務を3つのパターンで網羅する統合スキルです。特定地域の法律事務所競合調査、法制度の要件調査と適用分析、裁判例の網羅的調査と立証戦略の提言を行います。弁護士が依頼者説明や裁判所への提出資料の基礎として使えるレベルの正確性でレポートを作成します。

### (2) 内部ロジック

このスキルは3つの独立したパターンで構成されています：

#### パターン1: 法律事務所競合調査（3段階）

1. **弁護士ドットコムで事務所一覧取得**: 対象地域名で検索し、事務所名・所在地・弁護士数・取扱分野・特徴を取得します。弁護士ドットコム未登録の事務所が存在するため、弁護士会名簿（各地の弁護士会が公開している弁護士一覧）との照合が必要です。
2. **各事務所の公式HPから詳細確認**: 所属弁護士のプロフィール、取扱分野、アクセス情報、事務所の特徴を確認します。HP未設置の場合は弁護士会名簿やGoogleマップで代替調査します。
3. **不動産サイトから物件規模推定**: CBRE（不動産仲介大手の物件データベース）やオフィスナビでビル名と所在地を検索し、フロア面積を推定します。推定根拠がない場合は「不明（推定根拠なし）」と明記し、憶測では書きません。

**出力テンプレート**: Markdown表形式で「事務所名、地域、所在地、弁護士数、推定面積、取扱分野、出典」を含む一覧表を生成します。

#### パターン2: 法制度・要件調査 + 適用分析（5段階）

1. **事実関係の整理**: 依頼者から得た情報を構造化して整理します。
2. **制度の根拠法・概要の特定**: e-Gov法令検索（政府の法令データベース）や所管省庁の公式サイトで根拠法の条文番号、制度の趣旨、実施機関を確認します。
3. **要件の体系的整理**: 主体要件（誰が対象か）、客体要件（何が対象か）、手続要件（いつ、どこに、何を提出するか）、実体要件（どのような状態が必要か）の4カテゴリで全要件を網羅的に列挙します。
4. **本件事実関係との突合**: 各要件について充足見込みを◎（確実）、○（高い）、△（要確認）、✕（未充足）の4段階で評価し、判断マトリクス（評価表）を作成します。
5. **アクションプラン作成**: Phase分け（段階分け）した手続きの流れ、必要書類一覧、法定期限がある手続きの明示（★マーク）、依頼者に確認すべき追加情報を提示します。

#### パターン3: 裁判例・行政判断例の網羅的調査 + 立証戦略（5段階）

1. **判断基準の体系的整理**: 最高裁判例や行政報告から判断基準を確立し、大項目→小項目に階層化します。
2. **裁判例の網羅的収集**: 裁判所HP判例検索やWebSearchで広域検索し、最高裁判例（全件）、高裁判例（主要）、地裁判例（類似事実）を収集します。**捏造防止ルール**: WebSearchで確認できた裁判例のみを記載し、推測で裁判例を創作しません。
3. **裁判例の傾向分析**: 肯定例・否定例の共通要素、時系列での判断傾向の変化を分析します。
4. **本件への適用分析**: 判断基準の各要素について本件の事実関係を当てはめ、肯定(+)/否定(-)/要立証の評価をします。
5. **立証戦略の提言**: 揃えるべき証拠の優先順位（優先度A〜C）、提出資料の構成案、想定反論と対応方針、4段階アクションプランを提示します。

### (3) 使い方

パターンごとに起動タイミングが異なります：

- **パターン1**: 特定地域の法律事務所開業の市場分析が必要な時。複数地域を並列調査する場合は、地域別に複数の足軽に割り当てて同時実行できます。
- **パターン2**: 未払賃金立替払制度（会社が倒産した場合に未払賃金の一部を国が立て替える制度）や労災保険などの法制度が依頼者に適用できるか判断する時。
- **パターン3**: 労働者性（労働基準法の労働者に該当するか）や契約の成否などの法的論点について、裁判所や行政機関への提出資料の基礎を作成する時。

### (4) 実践例（弁護士業務での具体的ユースケース）

#### 実践例1: 倉敷市への法律事務所開業検討（パターン1）

弁護士として倉敷市内で法律事務所を開業する際、競合状況を把握する必要があります。このスキルを使って倉敷駅周辺・水島地区・岡山市南部の法律事務所19件を調査し、弁護士数の分布（単独弁護士が多い/大規模事務所は少ない）、取扱分野の空白地帯（企業法務に特化した事務所が少ない）、推定面積から参入機会（小規模事務所の多さ → 地域密着型で勝負できる）を特定できます。

#### 実践例2: 未払賃金立替払制度の適用可能性判断（パターン2）

依頼者が会社倒産により6ヶ月の給与未払いとなり、社長が連絡不通という状況で、未払賃金立替払制度を利用できるか調査します。このスキルで「事実上の倒産」認定の要件（事業活動停止、賃金支払能力喪失、再開見込みなし）を整理し、依頼者の状況を当てはめます。判断マトリクスで各要件の充足見込みを評価し、依頼者に「振込記録の収集」「未払賃金額の証明書類の準備」など具体的なアクションプランを提示できます。

#### 実践例3: リモートワーク労働者性の立証戦略（パターン3）

依頼者がリモートワークで働いていたが雇用契約書がなく、会社から「業務委託だった」と主張されているケースで、労働基準法の労働者性を立証する必要があります。このスキルで昭和60年労基法研究会報告（労働者性の判断基準を示した行政文書）の5要素（指揮監督下の労働、報酬の労務対償性など）を整理し、最高裁判例（旭紙業事件など）を収集します。振込記録が最強の間接証拠であることを示し、想定反論（「毎日出勤していない」→「リモートワークでも指揮監督は成立する」と援用する判例を提示）への対応方針を提言できます。

### (5) Tips/注意点

- **推定根拠の明記ルール（パターン1）**: 物件面積を推定する際、推定根拠がない場合は「不明（推定根拠なし）」と明記し、憶測で数値を書かないこと。依頼者への信頼性を損なわないため。
- **捏造防止ルール（パターン3）**: 裁判例はWebSearchで確認できたもののみを記載すること。「〜という判例があるはず」のような推測で裁判例を創作すると、裁判所への提出資料で致命的な誤りとなります。
- **品質レベルの違い**: パターン2は依頼者説明レベル（L2）、パターン3は提出資料レベル（L3）です。用途に応じて必要な調査深度が異なります。
- **並列実行の推奨（パターン1）**: 地域別に足軽を分担させることで、調査時間を1/3に短縮できます（cmd_116では19事務所を3足軽で30分で完了）。
- **他のツールとの使い分け**: 軽い概要調査ならWebSearchで十分ですが、弁護士が依頼者や裁判所に提示する正式なレポートが必要な場合はこのスキルを使います。

## 19. n8n-drive-notion-sync（Google Drive → Notion 自動連携スキル）— 16点

### (1) 概要

Google DriveにアップロードされたファイルをトリガーとしてNotion DBに自動登録する、n8nワークフロー構築パターンです。ファイル命名規則（ファイル名に日付や種別番号を含めるルール）を元に書面種別・証拠番号・案件名を自動抽出し、Notionで一元管理します。法律事務所の書面管理に特化していますが、不動産業や税理士事務所でも応用可能です。

### (2) 内部ロジック

#### ワークフロー全体構成（5段階）

1. **Google Drive Trigger（ファイル作成イベント監視）**: 特定フォルダ配下でファイルが作成されたことを5分間隔でポーリング（定期的にチェック）します。
2. **Function Node（ファイル名パース）**: 正規表現（パターンマッチング技術）で `YYYYMMDD_種別番号_書面名.拡張子` を分解し、書面種別（甲号証/乙号証/準備書面等）、証拠番号（甲1, 乙3-2 等）、案件ID（フォルダ名から抽出）を取得します。パース失敗時は `parseSuccess: false` を返します。
3. **IF Node（パース成功判定）**: パース成功なら書面DBに登録、失敗なら未分類DBに登録します。
4. **Notion Node（書面DB/未分類DB作成）**: 書面DBには書面名・種別・証拠番号・提出日・Driveリンク・案件とのリレーション（関連付け）を登録します。未分類DBにはファイル名・Driveリンク・エラー理由を登録します。
5. **Error Trigger（エラーハンドリング）**: パース失敗、Notion API Rate Limit（APIの呼び出し頻度制限。Notionは3回/秒まで）、案件ID未存在などのエラーをトリガーし、Google Chat通知やリトライ（再試行）を行います。

#### ファイル命名規則の具体例

| 書面種別 | ファイル名例 |
|---------|-------------|
| 甲号証 | `20260212_甲1_売買契約書.pdf` |
| 甲号証（枝番） | `20260212_甲1-2_契約書別紙.pdf` |
| 準備書面 | `20260220_準備書面1_原告第一準備書面.pdf` |
| 訴状 | `20260210_訴状_損害賠償請求.pdf` |

日付プレフィックス（先頭の日付）により、ファイルが時系列で自動的に並びます。

### (3) 使い方

- **Google Driveに書面をアップロードした時**: 命名規則に従ってファイル名を付けてアップロードすると、自動的にNotion書面DBに登録されます。手動でのNotion DB入力が不要になり、ヒューマンエラー（入力ミス）を防げます。
- **命名規則違反ファイルの場合**: パース失敗し、未分類DBに自動振り分けされます。週次レビュー（毎週確認）で未分類DBをチェックし、手動で正しいファイル名に修正します。
- **大量ファイルのバックフィル（過去ファイルの一括登録）**: 初期導入時に既存の数百〜数千ファイルを一括登録したい場合、Google Drive Node（List Files）で全ファイルを取得し、Split In Batches（バッチ処理）で10ファイルずつループ処理します。

### (4) 実践例（弁護士業務での具体的ユースケース）

#### 実践例1: 訴訟案件の書面管理自動化

裁判所から受け取った訴状や準備書面をGoogle Driveにアップロードする際、`20260210_訴状_損害賠償請求.pdf` という命名規則に従ってファイル名を付けます。すると自動的にNotion書面DBに「書面名: 損害賠償請求、種別: 訴状、提出日: 2026-02-10、Driveリンク: （URL）」が登録されます。案件フォルダ名（`2026_001_山田太郎_離婚`）から案件ID（`2026_001`）を抽出し、Notion案件マスタDBとリレーション（関連付け）されるため、案件ページから関連書面を一覧で確認できます。

#### 実践例2: 証拠管理の枝番対応

証拠が複数ページに分かれている場合、`20260212_甲1_契約書本体.pdf` と `20260212_甲1-2_契約書別紙.pdf` のように枝番（甲1-2）を付けます。このスキルは枝番にも対応しており、証拠番号フィールドに「甲1-2」が正確に抽出されます。裁判所への提出書類一覧を作成する際、Notion DBから証拠番号順にソート（並び替え）してエクスポートできます。

#### 実践例3: エラーハンドリングによる品質管理

事務スタッフがファイル名を誤って `契約書.pdf`（日付・種別番号なし）とアップロードした場合、パース失敗し未分類DBに登録されます。同時にGoogle Chat通知（「ファイル名が命名規則に従っていません: 契約書.pdf」）が届くため、即座に修正できます。週次レビューで未分類DBを確認し、命名規則違反のパターンを分析してスタッフ教育に反映できます。

### (5) Tips/注意点

- **命名規則の厳守**: このワークフローの成否は命名規則に100%依存します。ファイルアップロード前に必ずファイル名を確認する運用ルールが必須です。各種別ごとにファイル名テンプレート（`YYYYMMDD_甲XX_書面名.pdf`）を事務所内で共有しましょう。
- **Notion API Rate Limitへの対応**: NotionのAPIは3回/秒の厳格な制限があります。対策として、Notion Nodeの直前にWait Node（300ms待機）を配置し、大量ファイルのアップロード時はSplit In Batches（バッチ処理）で1バッチ3ファイルまでに制限します。
- **案件リレーションの精度**: 案件フォルダ名から案件IDを抽出する際、フォルダ名が変更されるとパースが失敗します。Notion案件DBに「Folder ID」プロパティを追加し、Google DriveのフォルダIDを直接保存することで精度を向上できます。
- **他業種への応用**: 不動産業（物件管理）、税理士事務所（顧客資料管理）、コンサルティング会社（プロジェクト資料管理）でも、ファイル命名規則とフォルダ構成を調整すれば同じパターンを適用できます。スキルファイルに応用例が記載されています。
- **セキュリティ**: Google Drive OAuth2認証は読取専用スコープ（`drive.readonly`）を推奨します。万が一n8nが侵害されても、Drive内のファイルが改ざんされるリスクを防げます。

## 20. n8n-automation-patterns（n8n自動化ワークフロー設計パターン）— 16点

### (1) 概要

n8nを使った業務自動化ワークフローの設計パターン集です。Gmail→Notion連携、Motion API連携、Webhook受信処理、定期実行レポート生成の4つの主要パターンを網羅し、認証設定・エラーハンドリング・セキュリティのベストプラクティス（最良の実践方法）を含みます。法律事務所やその他の士業（税理士・弁理士等）の業務自動化に特化したセキュリティ考慮事項も提供します。

### (2) 内部ロジック

#### パターン1: Gmail → Notion 自動連携

1. **Gmail Trigger（ポーリング5分間隔、ラベルフィルタ）**: 特定のGmailラベル（`law-firm/inbox` 等）のメールを5分ごとに取得します。
2. **IF（添付ファイル有無判定）**: 添付ファイルがあればGoogle Driveにアップロードし、Driveリンク URLを取得します。
3. **IF（件名パターンマッチ）**: 正規表現で件名から【案件名】プレフィックスを抽出し、案件紐付けします。紐付け失敗時は「未分類」として登録します。
4. **Notion（Database Item Create）**: メール件名・本文・送信元・添付ファイルリンク・案件とのリレーションをNotion DBに登録します。

#### パターン2: Motion API 連携

1. **Schedule Trigger（5分ごと）**: Motion APIから完了タスク（`status=COMPLETED`）を5分間隔で取得します。
2. **HTTP Request（GET Motion Tasks）**: API Key認証（`X-API-Key` ヘッダー）でMotion APIを呼び出します。Motion TeamプランはRate Limit（120 req/min）に余裕があり、5分間隔で安定運用できます。
3. **Notion Query Database**: Motion Task IDでNotion DBを検索し、既存レコードの有無を判定します。
4. **IF（既存レコードあり？）**: 既存レコードがあればUpdate、なければCreateします。一方向同期（Motion → Notion）を推奨します。

#### パターン3: Webhook受信 → DB登録

1. **Webhook Trigger（POST /webhook）**: Basic Auth（ユーザー名/パスワード認証）で外部サービスからのリクエストを受け付けます。
2. **Code Node（JSONスキーマバリデーション）**: 必須フィールドチェック、メールアドレス形式チェック等を行います。
3. **IF（バリデーション成功？）**: 成功ならNotion DB登録、失敗ならError Workflowでエラー通知します。
4. **HTTP Response（200 OK）**: Webhook送信元に成功レスポンスを返します。

#### パターン4: 定期実行 → レポート生成

1. **Cron Trigger（毎日9:00、毎週月曜9:00等）**: 指定した時刻にワークフローを起動します。
2. **Notion Query Database**: 先週の案件、未処理タスク等をフィルタして取得します。
3. **Code Node（データ集約・統計計算）**: 総案件数、完了数、進行中数、未着手数を計算します。
4. **Code Node（Markdown形式でレポート生成）**: Markdown形式で統計サマリーと詳細リストを生成します。
5. **Slack/Gmail（レポート送信）**: SlackチャンネルまたはGmailでレポートを送信します。

### (3) 使い方

- **Gmail→Notion連携**: 依頼者からのメールを自動的にNotion案件DBに記録したい時に使います。件名に【案件名】を付けるルールを依頼者に周知すれば、自動的に案件と紐付きます。
- **Motion API連携**: Motion（タスク管理・スケジューリングツール）で管理しているタスクの完了状況をNotionに同期し、一元管理したい時に使います。一方向同期（Motion → Notion）を推奨します。
- **Webhook受信**: 外部サービス（フォーム送信、Stripe決済完了等）のイベントを受信し、Notionに自動登録したい時に使います。
- **定期実行レポート**: 毎朝9時に前日の案件進捗サマリーをSlackに自動送信し、チーム全体で情報共有したい時に使います。

### (4) 実践例（弁護士業務での具体的ユースケース）

#### 実践例1: 依頼者メールの自動記録（Gmail→Notion）

依頼者からのメールを手動でNotionに転記する手間を削減したい場合、Gmail Triggerで特定ラベル（`法律相談`）のメールを監視し、Notion相談記録DBに自動登録します。件名に【離婚相談】と付けるルールを依頼者に周知すれば、案件紐付けが自動化されます。添付ファイル（診断書、契約書等）はGoogle Driveにアップロードされ、Driveリンクが記録されるため、Notionから直接ファイルにアクセスできます。

#### 実践例2: タスク管理の一元化（Motion→Notion）

Motionでタスクのスケジューリング（AIが最適な時間帯を提案）を行い、完了タスクの記録はNotionで一元管理したい場合、5分間隔でMotion APIから完了タスクを取得し、Notion進捗管理DBに同期します。Motionの「裁判準備」タスクが完了したら、自動的にNotion案件DBの「進捗ステータス」が更新されます。

#### 実践例3: 問い合わせフォームの自動登録（Webhook受信）

法律事務所のWebサイトに設置した問い合わせフォーム（Google Forms、Typeform等）の送信内容を自動的にNotion新規相談DBに登録します。Webhook TriggerでフォームからのPOSTリクエストを受信し、氏名・メールアドレス・相談内容をバリデーション後にNotionに記録します。必須フィールドが欠けている場合はエラー通知がSlackに飛び、スタッフが即座に対応できます。

#### 実践例4: 週次進捗レポートの自動送信（定期実行）

毎週月曜9時に先週の案件進捗レポートを自動生成し、Slackの#案件管理チャンネルに送信します。Cron Triggerで起動し、Notion DBから先週の案件（ステータスが「完了」「進行中」「未着手」）を取得し、統計計算（総案件数、完了率等）を行います。Markdown形式のレポートを生成し、Slack Nodeで送信します。弁護士が月曜朝に先週の進捗を一目で確認できます。

### (5) Tips/注意点

- **セルフホスト vs Cloud の判断**: 弁護士業務では守秘義務（依頼者の情報を厳格に保護する義務）があるため、データ保管場所が重要です。n8n Cloud（月額約3,000円）はEU域内（ヨーロッパ）にデータが保管されます。日本国内保管が必須の場合はセルフホスト（自分でサーバーを立てる方式、月額750〜1,500円のVPSで可能）を選択します。
- **弁護士情報セキュリティ規程（2024年6月施行）への対応**: セルフホストで日本国内VPS（ConoHa、さくらのクラウド等）を使用し、リバースプロキシ（Nginx + Let's Encrypt）でHTTPS化（通信の暗号化）します。`N8N_ENCRYPTION_KEY`（認証情報の暗号化キー）を厳重管理し、流出すると全認証情報が漏洩します。
- **Notion API Rate Limitへの対策**: NotionのAPIは3回/秒の厳格な制限があります。Wait Node（300ms待機）をNotion Nodeの直前に挿入し、Exponential Backoff（リトライ間隔を段階的に延長: 1s → 2s → 5s）を設定します。
- **Motion API Rate Limitとプラン選択**: Motion IndividualプランはRate Limitが12 req/min（同期間隔10分以上が必要）、TeamプランはRate Limitが120 req/min（5分間隔で安定運用可能）です。業務用途ならTeamプランを推奨します。
- **双方向同期のリスク**: Motion ↔ Notion の双方向同期は競合リスク（同じタスクを両側で編集した場合のデータ整合性の問題）が高いため、一方向同期（Motion → Notion）を強く推奨します。MotionのAIスケジューリング機能はMotion側で完結し、Notionは案件管理・資料紐付けに特化させるのが実用的です。
- **委任契約書への記載**: 依頼者に対して「当事務所は、セキュリティ対策を講じたワークフロー自動化ツール（n8n）を利用して事件情報を管理します。データは日本国内のサーバーに暗号化保存されます。」と明記することで、透明性を確保できます。

## まとめ

この3件のSランクプロジェクトスキルは、いずれも法律事務所業務の自動化・効率化に直結するツールです。

- **legal-office-research**: 弁護士が依頼者説明や裁判所提出資料を作成する際の調査業務を大幅に効率化します。
- **n8n-drive-notion-sync**: 書面管理の手動入力を完全自動化し、ヒューマンエラーを防ぎます。
- **n8n-automation-patterns**: Gmail、Motion、Webhookなどを連携し、情報の一元管理と定期レポート生成を実現します。

これらのスキルを組み合わせることで、弁護士が本来の業務（法律相談、訴訟対応）に集中できる環境を構築できます。
